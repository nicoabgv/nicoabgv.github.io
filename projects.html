Case study 01

MesasYa — SaaS de reservas multi-sede

MesasYa es un SaaS en producción que digitaliza la gestión de reservas para restaurantes con múltiples sedes. Reúne planos interactivos, reglas personalizables y notificaciones en tiempo real para que cada servicio funcione sin sorpresas.

El backend basado en Flask, Celery, MySQL y Docker mantiene despliegues reproducibles y seguros. Cada restaurante configura zonas, servicios, idiomas y workflows específicos sin perder estabilidad ni trazabilidad.

Impacto operativo

  • Planos de sala configurables con reglas por sede, zonas y eventos especiales.
  • Asignación automática de mesas y notificaciones en tiempo real para staff y clientes.
  • Exportaciones y métricas listas para BI que acompañan las decisiones del negocio.

Stack

  • Flask · Python
  • Celery + Redis
  • MySQL administrado
  • Docker Compose

Próximos pasos

Integración de informes automáticos y mejoras en la experiencia del gestor para acelerar la toma de decisiones.

Espacio ampliable

Material listo para incluir mapas de calor, modelos de demanda y documentación de despliegue.

Case study 02

Kobra — Plataforma financiera para Cybersur

Kobra es la solución a medida que mantiene la operativa financiera de Cybersur. Automatiza facturación, cobros y recordatorios desde un backend robusto en FastAPI, Celery, MySQL y Docker.

Los cronjobs programados gestionan exportaciones, limpiezas de datos y conciliaciones automáticas, mientras que los dashboards internos ofrecen visibilidad en tiempo real sobre el flujo de caja y SLA por cliente.

Resultados clave

  • Workflows de avisos y recordatorios que reducen el tiempo de seguimiento de facturas.
  • Cronjobs trazables para exportaciones contables y rutinas de limpieza de datos.
  • Dashboards internos con indicadores de cobro y alertas proactivas.

Stack

  • FastAPI · Python 3.11
  • Celery + Redis
  • MySQL administrado
  • Docker Compose

Roadmap

Integraciones con ERPs externos y ampliación de reportes financieros internos.

Material adicional

Preparando diagramas de flujo, auditorías de seguridad y documentación para nuevos módulos.

Case study 03

BeatList — Inteligencia musical para power users

Proyecto oficial Spotify

BeatList es la plataforma para melómanos, curadores y creadores de contenido que buscan superar las limitaciones del gestor estándar de Spotify. Centraliza administración avanzada, descubrimiento y analítica en una experiencia fluida.

El backend en Flask gestiona la autenticación OAuth 2.0 aprobada por Spotify y accede a datos extendidos mediante su API oficial. El frontend en JavaScript dinamiza dashboards, recomendaciones y un reproductor web con animaciones suaves.

Experiencia destacada

  • Listas inteligentes basadas en reglas y métricas como energía o bailabilidad.
  • Dashboard con tendencias temporales y hábitos de escucha segmentados por artistas y géneros.
  • Reproductor web con animaciones personalizadas y continuidad entre sesiones gracias a MySQL y Redis.

Stack

  • Flask · Python
  • Spotify API · OAuth 2.0
  • JavaScript modular
  • MySQL + Redis

Roadmap

Modo colaborativo, exportación de insights y app móvil como siguientes iteraciones.

Material extendido

Espacio reservado para flujos de onboarding, mockups de dashboards y métricas avanzadas.
