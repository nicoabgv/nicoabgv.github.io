Case study 01

MesasYa — SaaS de reservas multi-sede

MesasYa es un que digitaliza la gestión de reservas para restaurantes con múltiples sedes. Reúne planos interactivos, reglas personalizables y notificaciones en tiempo real para que cada servicio funcione sin sorpresas.

El backend basado en utiliza Docker Compose y buenas prácticas de seguridad. Cada restaurante configura zonas, servicios, idiomas y workflows específicos sin perder estabilidad ni trazabilidad.

Acceso directo

→ MesasYa – Plataforma SaaS de gestión de reservas para restaurantes.

Impacto operativo

  • Planos de sala configurables con reglas por sede, zonas y eventos especiales.
  • Asignación automática de mesas y notificaciones en tiempo real para staff y clientes.
  • Exportaciones y métricas listas para BI que acompañan las decisiones del negocio.

Stack

  • Flask · Python
  • Celery
  • MySQL administrado
  • Docker Compose

Próximos pasos

Integración de informes automáticos y optimización de la experiencia de gestión.

Esquema técnico 
Reservas web / staff
    ↓
Flask API → MySQL
    │
    ├─ Celery workers → Correos/SMS
    └─ Docker Compose coordina servicios

Case study 02

OrderYa — Gestión integral de pedidos

OrderYa elimina el caos de la gestión de pedidos entre obradores y puntos de venta. Sustituye hojas de cálculo y mensajes dispersos por una plataforma centralizada donde cada tienda realiza sus pedidos de forma intuitiva y el obrador visualiza la demanda consolidada.

Desarrollado con y renderizado server-side (Jinja2) para garantizar velocidad y simplicidad, el sistema gestiona catálogos complejos, listas de precios por cliente y generación automática de órdenes de reparto.

Acceso directo

→ OrderYa – Centralización y gestión de producción.

Funcionalidades clave

  • Portal de tiendas con histórico y estados en tiempo real.
  • Escaneo de código de barras para agilizar la creación de pedidos.
  • Creación de plantillas, gestión de productos y exportación de datos.

Stack

  • FastAPI + Jinja2
  • MySQL 8.0
  • Docker Compose
  • Nginx Proxy

Estado

En producción. Gestionando pedidos diarios para obradores reales.

Esquema técnico 
Tiendas
    ↓
Nginx → FastAPI
    ↓
MySQL (Persistencia)
    ↓
Producción (Listas PDF/Vista)

Case study 03

Trato — CRM ágil para equipos comerciales

Trato centraliza presupuestos, seguimientos y recordatorios en un CRM ligero pensado para equipos comerciales. Con un backend basado en ejecutado con Uvicorn, prioriza la velocidad de respuesta y la claridad de cada oportunidad.

Las empresas acceden mediante códigos de invitación y gestionan pipelines con estados personalizables, timeline de interacciones y avisos automáticos. Las tareas en Celery coordinan recordatorios, envíos de emails y resúmenes diarios desde el VPS principal.

Acceso directo

→ Trato – CRM ágil para presupuestos, seguimientos y recordatorios.

Impacto

  • Altas controladas de empresas y usuarios mediante códigos de invitación.
  • Seguimientos con recordatorios automáticos y timeline centralizado por oportunidad.
  • Despliegue contenerizado en el VPS principal con backups y monitoreo de tareas Celery.

Stack

  • FastAPI · Uvicorn
  • Celery
  • MySQL
  • Docker Compose

Estado

Producción en VPS principal con monitoreo básico y backups diarios.

Esquema técnico 
Empresas invitadas
    ↓
FastAPI + Uvicorn → MySQL
    │
    ├─ Celery workers → Recordatorios y emails
    └─ Docker Compose → VPS principal

Case study 04

Kobra — Plataforma financiera para Cybersur

Kobra es la solución a medida que mantiene la operativa financiera de Cybersur. Automatiza facturación, cobros y recordatorios desde un backend robusto en .

Los cronjobs programados gestionan exportaciones, limpiezas de datos y conciliaciones automáticas. Los paneles internos priorizan métricas de cobranza y SLA por cliente para que el equipo financiero tenga claridad diaria.

Resultados clave

  • Workflows de avisos y recordatorios que reducen el tiempo de seguimiento de facturas.
  • Cronjobs trazables para exportaciones contables y rutinas de limpieza de datos.
  • Paneles internos con indicadores de cobro y alertas sobre vencimientos.

Stack

  • FastAPI · Python 3.11
  • Celery
  • MySQL administrado
  • Docker Compose

Roadmap

Ampliación de reportes financieros internos y automatización de conciliaciones.

Esquema técnico 
Equipo Cybersur
    ↓
FastAPI API → MySQL
    │
    ├─ Celery workers → Emails y recordatorios
    └─ Cronjobs → Reportes contables

Case study 05

BeatList — Inteligencia musical para power users

Spotify Approved

BeatList es la plataforma para melómanos, curadores y creadores de contenido que necesitan más herramientas que el gestor estándar de Spotify. Centraliza administración avanzada, descubrimiento y analítica en una experiencia fluida.

El backend en Flask gestiona la autenticación aprobada por Spotify, coordina análisis pesados con Celery y accede a datos extendidos mediante la . El frontend en JavaScript dinamiza dashboards, recomendaciones y un reproductor web con animaciones suaves.

Acceso directo

→ BeatList – Estadísticas y gestión avanzada de playlists de Spotify.

Experiencia destacada

  • Listas inteligentes basadas en reglas y métricas como energía o bailabilidad.
  • Dashboard con tendencias temporales y hábitos de escucha segmentados por artistas y géneros.
  • Análisis gestionados con Celery para procesar datos en segundo plano y ofrecer métricas precisas sin afectar la experiencia del usuario.

Stack

  • Flask · Python
  • Spotify API · OAuth 2.0
  • JavaScript modular
  • Celery para análisis asíncronos

Posibles evoluciones

Añadir modo colaborativo o exportación de insights en futuras versiones.

Esquema técnico 
Usuarios Spotify

Flask API (OAuth 2.0)
    ↓
Spotify API oficial → Datos enriquecidos
    ↓
Celery workers → Análisis pesados y métricas
    ↓
Frontend JS → Dashboards + reproductor
