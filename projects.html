Case study 02

MiCartaYa — Ecosistema para restaurantes

MiCartaYa redefine la interacción en el restaurante mediante una plataforma SaaS de menús digitales y pedidos autogestionados. Resuelve el cuello de botella de la toma de pedidos manual y garantiza que la información de la carta esté siempre actualizada.

El núcleo del proyecto es un sistema de WebSockets de alta disponibilidad que conecta a los comensales con el dashboard del restaurante y el hardware de impresión, eliminando errores de transcripción y optimizando los tiempos de servicio.

Acceso directo

→ MiCartaYa – Pedidos en tiempo real y menú digital multiidioma.

Impacto en sala

  • Cartas interactivas multiidioma con actualización instantánea y control de stock.
  • Comunicación mediante WebSockets para recepción de pedidos en menos de 1 segundo.
  • Integración de impresión térmica remota para comandas automáticas en cocina.

Stack

  • FastAPI · Python
  • WebSockets (FastAPI)
  • MySQL · SQLAlchemy
  • Docker · Nginx

Roadmap

Módulo de pagos integrados y fidelización de clientes.

Esquema técnico 
Comensal (App) ↔ WebSockets ↔ Backend (FastAPI)
                                    ↓
                                MySQL DB
                                    ↓
                          Impresión Térmica TPV

Case study 04

Trato — CRM ágil para equipos comerciales

Trato centraliza presupuestos, seguimientos y recordatorios en un CRM ligero pensado para equipos comerciales. Con un backend basado en FastAPI ejecutado con Uvicorn, prioriza la velocidad de respuesta y la claridad de cada oportunidad.

Las empresas acceden mediante códigos de invitación y gestionan pipelines con estados personalizables, timeline de interacciones y avisos automáticos. Las tareas en Celery coordinan recordatorios, envíos de emails y resúmenes diarios desde el VPS principal.

Acceso directo

→ Trato – CRM ágil para presupuestos, seguimientos y recordatorios.

Impacto

  • Pipeline acelerado: visualización clara del ciclo de vida de cada presupuesto.
  • Recordatorios inteligentes: Celery gestiona el 'follow-up' para que ninguna oferta se pierda.
  • Control de acceso seguro mediante sistema robusto de códigos de invitación.

Stack

  • FastAPI · Uvicorn
  • Celery
  • MySQL
  • Docker Compose

Estado

Producción en VPS principal con monitoreo básico y backups diarios.

Esquema técnico 
Empresas invitadas
    ↓
FastAPI + Uvicorn → MySQL
    │
    ├─ Celery workers → Recordatorios y emails
    └─ Docker Compose → VPS principal

Case study 05

Kobra — Plataforma financiera para Cybersur

Kobra es la solución a medida que mantiene la operativa financiera de Cybersur. Automatiza facturación, cobros y recordatorios desde un backend robusto en FastAPI.

Los cronjobs programados gestionan exportaciones, limpiezas de datos y conciliaciones automáticas. Los paneles internos priorizan métricas de cobranza y SLA por cliente para que el equipo financiero tenga claridad diaria.

Resultados clave

  • Mejora del flujo de caja mediante flujos de avisos automáticos a clientes.
  • Auditoría completa: trazabilidad total de cobros y estados financieros.
  • +20% de eficiencia operativa en el equipo financiero al eliminar tareas repetitivas.

Stack

  • FastAPI · Python 3.11
  • Celery
  • MySQL administrado
  • Docker Compose

Próximos pasos

Automatización de reportes financieros y conciliaciones.

Esquema técnico 
Operativa Cybersur
    ↓
FastAPI → MySQL
    │
    ├─ Celery → Emails
    └─ Cronjobs → Reportes

Case study 06

BeatList — Analítica avanzada de Spotify

Spotify Approved

BeatList acompaña a curadores y creadores que necesitan superar el gestor estándar de Spotify. Conecta la Spotify API con análisis avanzados y experiencias interactivas aprobadas por el programa Spotify for Developers.

El backend en Flask con OAuth 2.0 obtiene datos extendidos y coordina análisis con Celery para generar métricas sin bloquear a los usuarios. El frontend ofrece reproductores, dashboards y herramientas de mezcla pensadas para profesionales y comunidades.

Acceso directo

→ BeatList – Estadísticas y gestión avanzada de playlists de Spotify.

Resultados

  • Smart data cleanup: duplicate debugging algorithms and audio filters.
  • Guaranteed security: official OAuth 2.0 authentication validated by Spotify.
  • High-impact UX: interactive dashboards with real-time audio analytics.

Stack

  • Flask · Python
  • Spotify API · OAuth 2.0
  • JavaScript modular
  • Celery para análisis asíncronos

Próximos pasos

Modo colaborativo y exportaciones avanzadas en la hoja de ruta.

Esquema técnico 
Usuarios\n    ↓\nFlask API (OAuth Spotify)\n    ↓\nCelery workers → Análisis pesados e insights\n    ↓\nDashboards + Reproductor

Case study 07

Ponme — Asistente de moda inteligente

Ponme democratiza el uso de la IA generativa para resolver el dilema diario de 'qué me pongo'. Integra la API de Gemini para entender no solo qué ropa tienes, sino cómo combinarla según el contexto.

La aplicación procesa imágenes localmente para eliminar fondos y utiliza modelos multimodales para extraer metadatos de las prendas (color, tejido, estilo), permitiendo crear un 'armario digital' en minutos.

Acceso directo

→ Ponme – Tu armario inteligente potenciado por IA.

Características IA

  • Etiquetado automático y categorización de prendas mediante IA multimodal (Google Gemini).
  • Algoritmo de sugerencias que optimiza el uso de la ropa según contexto real y estado de las prendas.
  • Pipeline de procesamiento de imagen eficiente para dispositivos móviles.

Stack

  • FastAPI · Python
  • React · Tailwind
  • Google Gemini API
  • PostgreSQL · Docker

Próximos pasos

Sugerencias de compra basadas en huecos del armario e inteligencia predictiva.
